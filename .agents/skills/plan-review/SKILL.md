# plan-review

Use this skill before implementation.

1. Check whether the plan has `Why`, `Non-goals`, `Scope`, `Acceptance Criteria`, `Verification`, and `Rollback`.
2. Look for hidden schema changes, missing auth boundaries, and unowned follow-up work.
3. Reject plans that cannot be evaluated mechanically.
